Discipline Best Practices
As the resident source control guy for our game it is my responsibility to keep our repository in working order and to solve problems as they come up. Our team uses git as our primary source control management system so I try to keep current with updates. 

Lately git has implemented a feature called LFS (large file storage). Git has a limit of 50mb for a single file which hurts our ability to save large files like .dll, .ai, and .psd files. Even files that are under the 50mb limit but are rather large can impact our entire team when it comes to pulling updates, especially when not every member of the team needs these changes. Thankfully, LFS helps fix this problem by allowing us to set specific files or file types to track using LFS. LFS specializes in uploading/downloading and storing these files and doesn't burden the whole team with updates to these files unless they are the owner of the content or request it. This has made pulling and cloning our repository a lot easier.

This has really come in handy for our artist and for our sound designer who deal with large files. I've taught them about LFS and how to track files using this system.
